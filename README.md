# BsimpleHires - B站「高解析度音质」一键转换工具

![Logo](assets/logo.ico)

## 一句话介绍
一个为了让 B 站音乐、翻唱、Live、ASMR、直播切片的音质更好一点，而诞生的“一步到位”工具。它能将普通音视频文件一键转换为更符合 B 站「Hi-Res 音质」上传要求的格式。

> 如果你是想在 B 站上传音乐、翻唱、Live、ASMR、直播切片，却苦于音频总被压缩的 UP 主或乐迷，这个工具就是为你准备的。

## 我为什么做这个
1. B 站是个很好用的“音乐播放器”，但普通上传流程下音频压缩比较重。
2. B 站给高音质内容留了更高规格的上传入口，也就是「Hi-Res 音质」。
3. 但这套要求对普通用户并不友好，尤其是 `24bit PCM`、`48kHz+` 这些条件，很多常见剪辑工具不会直接帮你处理好。
4. 这个项目的目标，就是把原本繁琐的处理流程变成尽量直接的一键操作。

## 核心特点
- 智能达标：自动将音频转换为 `PCM 24bit`，并按规则处理采样率，输出更符合 B 站 Hi-Res 目标规格。
- 极速转换：视频流直接复制，只处理音频，速度明显快于整段重编码，同时保留原视频画质。
- 真正可用的批量处理：支持多文件并发转换，并显示总进度和每个活跃文件的独立进度。
- 可取消：转换过程中可以取消任务，已完成文件保留，未完成输出会清理。
- 设置持久化：支持默认输出目录、最大并发数、命名策略、自定义 FFmpeg 目录、完成后自动打开输出目录。
- 日志可追踪：运行日志会写入 `logs/bsimplehires.log`，便于排错。

## 快速开始
如果你只是想直接使用：

1. 准备好 `ffmpeg` 和 `ffprobe`。  
   优先级如下：
   - 系统 PATH 中的 `ffmpeg/ffprobe`
   - 程序同级目录下的 `ffmpeg/` 文件夹
   - 设置里指定的自定义 FFmpeg 目录
2. 运行 `BsimpleHires.exe` 或开发环境下执行 `python main.py`
3. 导入文件并点击开始转换

转换完成后，你会得到一个带 `_bsimple.mov` 后缀的输出文件，可直接用于后续上传。

## 详细使用指南
### 1. 导入文件
点击“导入文件”按钮，选择你的 MP4、MKV、WAV、MP3 等常见格式文件。

### 2. 查看与选择
文件列表会显示时长、大小、分辨率、视频格式、音频格式、采样率等信息。默认勾选全部文件，也可以只保留需要转换的项。

### 3. 开始转换
点击“开始转换”，选择输出目录后确认执行。当前版本支持：

- 总进度条
- 每个正在转换文件的独立进度条
- 转换中取消
- 部分失败时保留成功结果并提示错误

### 4. 获取结果
输出文件默认命名为：

```text
原文件名_bsimple.mov
```

如果同名文件已存在：

- 默认策略：自动生成 `_2`、`_3` 这样的递增文件名
- 可在设置中改为直接覆盖同名输出

## 技术原理
为了让文件尽量符合目标规格，程序会自动做这些处理：

| 处理项目 | 具体操作 | 目的 |
| :--- | :--- | :--- |
| 音频格式 | 转换为 `PCM 24bit` (`pcm_s24le`) | 满足高规格上传对位深的要求 |
| 采样率 | `<48kHz` 升至 `48kHz`；`>=48kHz` 保持原样 | 尽量满足目标要求，同时避免无意义降采样 |
| 视频流 | `copy` 直接复制 | 保持画质，减少转换时间 |
| 容器格式 | 输出为 `MOV` | 与目标上传场景保持兼容 |

## 常见问题
### Q1: 为什么转换后的文件会变大？
A: 因为音频会被转换成无损的 `PCM 24bit`。体积变大是正常结果，这也是更高音质的代价之一。

### Q2: 我用剪映、必剪导出的视频，还能再处理吗？
A: 可以。这正是这个工具的典型使用场景之一。先正常剪辑导出，再用本工具处理最终成品即可。

### Q3: 支持 Mac 或 Linux 吗？
A: 当前仍以 Windows 为主要运行平台。
